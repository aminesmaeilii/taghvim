# برنامه پیاده سازی تقویم محتوا

## وضعیت اولیه

پوشه پروژه در زمان شروع خالی بود؛ بنابراین کد یا معماری قابل حفظی وجود نداشت. این پروژه از ابتدا با Tauri 2، React، TypeScript و SQLite طراحی می شود. لایه نما هرگز مستقیما با SQLite صحبت نمی کند و فقط از فرمان های تاییدشده Tauri استفاده می کند.

## مسیر اجرا

1. **زیرساخت و پوسته برنامه**: پیکربندی Tauri/Vite، توکن های طراحی، فونت محلی، RTL، مسیریابی، میان برها، تم و Error Boundary.
2. **داده و دامنه**: مدل های تایپ شده، اعتبارسنجی Zod، migration های SQLite، repository های Rust و داده های پایه قابل تنظیم.
3. **محتوا و تقویم**: فرم سریع و کامل، پیش نمایش و ویرایش، تقویم جلالی ماه/هفته/روز/فهرست، فیلتر، جستجو، جابه جایی و undo.
4. **گردش کار و برنامه ریزی**: برد کانبان، کمپین، ایده، قالب، ستون محتوا و گزارش های کاربردی.
5. **یکپارچگی دسکتاپ**: import/export/backup، انتخاب فایل بومی، notification های با رضایت کاربر و پیکربندی بسته بندی Tauri.
6. **کیفیت**: تست های واحد منطق جلالی و اعتبارسنجی، بررسی TypeScript، lint، build فرانت اند و مرور RTL/دسترس پذیری.

## تصمیم های معماری

- SQLite در `src-tauri` منبع اصلی داده است؛ `src/services/content-repository.ts` تنها مرز دسترسی نمای React است.
- در اجرای مرورگر توسعه، یک repository موقت در حافظه صرفا برای پیش نمایش UI استفاده می شود و هیچ داده هسته ای در localStorage ذخیره نمی شود.
- تنظیمات ظاهری کوچک در localStorage نگهداری می شوند؛ داده های محتوا، کمپین و ایده تنها از مسیر SQLite می آیند.
- تاریخ های ذخیره شده ISO/Gregorian هستند؛ تبدیل و نمایش جلالی در یک utility واحد انجام می شود تا query و index دیتابیس دقیق بماند.
- سرویس های sync و authentication در مرزهای جدا پیش بینی شده اند اما در نسخه آفلاین نخست فعال نیستند.

## محدوده نسخه نخست

نسخه نخست بر برنامه ریزی آفلاین حرفه ای تمرکز دارد: محتوا، تقویم، گردش کار، کمپین، ایده، قالب، گزارش، export/import و backup. همگام سازی ابری، همکاری هم زمان و ارسال خودکار به شبکه های اجتماعی به دلیل نیاز به سرویس و مجوز بیرونی عمدا به نسخه بعد موکول شده اند.

## ریسک محیط اجرا

برای اجرای `tauri dev` و ساخت installer، Rust/Cargo و پیش نیازهای Tauri باید روی سیستم نصب باشند. این محیط در زمان بررسی Cargo نداشت؛ بنابراین بررسی build بومی فقط پس از فراهم شدن آن ابزارها ممکن است.

## وضعیت اجرای این برنامه

- **انجام‌شده:** پوستهٔ RTL، فونت محلی، routing، داشبورد، تقویم چهارحالته، فرم مرحله‌ای محتوا، فهرست، برد گردش کار، کمپین، ایده، قالب، گزارش، تنظیمات، فیلتر و جستجو، drag-and-drop با Undo، خروجی JSON/CSV، چاپ، validation و تست‌های جلالی/Zod.
- **انجام‌شده:** fallback مرورگر با IndexedDB برای حفظ داده در refresh، مرز `ContentRepository`، Tauri 2 config، migration اولیهٔ SQLite، commandهای IPC و مجوزهای محدود.
- **در انتظار محیط build:** اجرای `tauri dev` و تولید installer به Rust/Cargo، WebView2 و toolchain بومی سیستم‌عامل نیاز دارد؛ این ابزارها در محیط audit حاضر نیستند.
- **موکول‌شده:** sync ابری، حساب کاربری، همکاری هم‌زمان، اتصال مستقیم به شبکه‌های اجتماعی و گزارش‌گیری native notification.
